# 2.5D Battle UI — Design

Give the battlefield depth: tilt the board like a tabletop and make unit
tokens stand upright on it, HoMM-style cardboard cutouts.

## Approach: CSS perspective tilt

Chosen over a true isometric (diamond) rebuild and a canvas renderer because it
keeps the shipped DOM grid — layout, clicks, highlights, aria-labels — fully
intact. Browsers hit-test 3D-transformed elements natively, so interactions
need no changes.

- A wrapper around the grid gets `perspective` (~1100px, origin above center).
- The grid itself gets `transform: rotateX(~48deg)` with
  `transform-style: preserve-3d` propagated down through the cell buttons.
- Each occupied cell wraps its `UnitToken` in a "standing" div:
  `rotateX(-48deg)` with `transform-origin: bottom`, bottom-anchored and
  slightly taller than the cell, so the token rises out of the board plane.
  An elliptical shadow under the token grounds it.
- `preserve-3d` gives correct depth sorting for free: nearer rows occlude
  farther ones.
- Turn-bar tokens stay flat (the standing wrapper lives in `BattleGrid`, not
  in `UnitToken`, which stays presentation-agnostic).
- The victory/defeat overlay sits outside the perspective wrapper, unaffected.
- Tilt foreshortens board height; compensate with container sizing/margins,
  tuned by screenshot.

## Testing

No new logic — presentation only. Existing Vitest suite must stay green;
visual verification by driving a battle in headless Chrome and eyeballing
screenshots (move highlights, standing tokens, occlusion, overlay).
